[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the workflow architecture. When a user submits a request, it first goes to our Security Checkpoint. This node detects prompt injections, redacts PII, and flags fraudulent costs. If clean, the request goes to the Event Orchestrator. 
The Orchestrator delegates tasks to two specialized LlmAgents: the RSVP Manager, which tracks guests and queries our local MCP catering options, and the Expense Calculator, which calculates exact cost splits. 
Before anything is finalized, we pass through a Human Review node using the ADK's RequestInput. This keeps a human-in-the-loop, allowing the coordinator to inspect and approve the plan before the Final Output is generated.

[3:00 — SECURITY & MCP]
The security checkpoint is the shield of this application. It blocks prompt injection tricks and PII instantly. For example, if someone inputs an email address or tries to claim a $6,000 expense, the checkpoint flags it, writes a structured JSON audit log, and stops the execution before calling the LLM. 
Our MCP server exposes three tools: catering retrieval, venue details, and automatic invitation email generation, providing the LLM with real-time data.
